# グローバルナビゲーション（グロナビ）要件定義書

## 1. 目的
Rainy・Perch LP（ランディングページ）内の各セクションへ、ユーザーがスムーズにアクセス・回遊できるようにするためのナビゲーションメニューを実装する。

## 2. ターゲット
- PCユーザー（マウス操作）
- モバイルユーザー（タップ・スワイプなどのタッチ操作）

## 3. 機能要件
1. **ページ内リンクのスムーズスクロール**
   - メニュー項目をクリック/タップすると、対応するセクションへ滑らかにスクロール（Smooth Scroll）して移動する。
2. **モバイル対応（右から展開するフルスクリーンドロワー）**
   - モバイル表示時は、画面右上にハンバーガーアイコンを表示。
   - タップすると、右側から画面全体（全画面）を覆う形でドロワーメニューが引き出される。
   - リンクを選択、または背景・閉じるボタンをタップすると自動でメニューが閉じる。
3. **上部固定（Fixed Nav）とスクロール連動による高さ変化**
   - ナビゲーションは常に画面上部に固定されて表示（隠れない）。
   - ページ最上部（スクロール0）の時はデフォルトの高さ。
   - 下へスクロールした際は、ナビゲーションの縦幅（高さ）がほんの少し狭まり、スッキリした見た目に変化する（最上部に戻ると元の高さに戻る）。

## 4. デザイン・UI要件
- **配置**:
  - 全体レイアウト：左詰めに「ロゴ」、右詰めに「メニュー項目（PC） / ハンバーガーアイコン（モバイル）」を配置。
  - ロゴ画像：制作中のため、一旦サイトのファビコン（または既存のアイコン画像）を仮置きする。
- **カラースキーム**:
  - 本LPのカフェの雰囲気（クリーム色、ブラウン）に合わせる。
  - スクロール時は背景 `bg-cream/90` に `backdrop-blur`（すりガラス効果）をかけて下のコンテンツがうっすら透けるようにし、圧迫感を減らす。

## 5. ナビゲーション項目（英語表記）
以下のように各セクションに対応するリンクを設置する。
1. **Top** (Hero セクションへ戻る)
2. **Concept** (Concept セクション)
3. **Staff** (Staff セクション)
4. **Gallery** (Gallery セクション)
5. **News** (News セクション)
6. **VRC Group** (VRC Group セクション)
7. **Join** (HowToJoin セクション 事前予約など)

## 6. 実装アプローチ
- 新規コンポーネント `GlobalNav.tsx` を作成し、`App.tsx` 内の全体を囲むコンテナの最も上部（`fixed z-50 w-full`）としてマウントする。
- スクロールイベントを監視し、`window.scrollY > 0` かどうかで、ナビゲーションの高さを CSS の transition で滑らかに切り替える。
- 各 `<section>` に `id="concept"`, `id="staff"` などの ID を付与し、アンカーリンクとCSSの `scroll-behavior: smooth;` を利用してページ内リンクを実装する。
