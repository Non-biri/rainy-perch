# カフェLPサイト作成 要件定義書 (ドラフト)

## 1. プロジェクト概要
**目的**: 新規（または既存）カフェの魅力を伝え、来店を促進するためのランディングページ（LP）を作成する。
**ゴール**: 
- 店舗の雰囲気やコンセプトをユーザーに直感的に伝える。
- メニューや場所、営業時間などの基本情報へのアクセスを容易にする。
- 来店予約や問い合わせ（必要な場合）への動線を確保する。

## 2. ターゲットユーザー
- **メイン層**: VRや仮想世界に興味のある若者
- **ニーズ**: 
    - 仮想世界でのおしゃれな空間でくつろぎたい
    - 視覚的に楽しめるメニューを見たい
    - 仮想世界での交流を楽しみたい

## 3. デザインコンセプト
- **キーワード**: [例: 落ち着いた、モダン、温かみのある、ナチュラル、レトロ]
- **カラーパレット**: 
    - ベースカラー: ブラウン
    - メインカラー: クリーム
    - アクセントカラー: 緑
- **フォント**: 視認性が高く、お店の雰囲気に合ったもの（例: 明朝体で高級感を出す、ゴシック体で親しみやすさを出す）

## 4. サイト構成 (Sitemap)
シングルページのLP構成を想定。

1.  **ヒーローエリア (Hero Section)**
    - ファーストビューで店舗の雰囲気が伝わる高品質なメインビジュアル（画像）。
    - キャッチコピー。
2.  **コンセプト (Concept)**
    - 店舗のこだわり、ストーリー、店主の想い。
3.  **メニュー (Menu)**
    - おすすめメニューの紹介（画像のみ）。
    - 価格や説明の記載は不要。
4.  **スタッフ紹介 (Staff)**
    - スタッフの紹介エリア。
5.  **ギャラリー / 雰囲気 (Atmosphere)**
    - 内観、外観、スタッフの様子などがわかる写真スライドショーまたはグリッド。
6.  **ニュース / お知らせ (News)**
    - X (旧Twitter) の埋め込み。
7.  **参加方法 (How to Join)**
    - 仮想世界（RainyPerch）への参加方法・アクセス方法。
7.  **フッター (Footer)**
    - コピーライト、SNSリンク（Xのみ）。

## 5. 機能要件
- **レスポンシブ対応**: スマートフォン、タブレット、PCそれぞれのデバイスで最適化表示。
- **SNS連携**: Instagramフィードの埋め込みは不要。
- **お問い合わせフォーム**: Xのアカウントへ誘導。
- **SEO対策**: 基本的なメタタグ設定、OGP設定。
- **高速化**: 画像の軽量化、読み込み速度の最適化。

## 6. 技術スタック (推奨)
- **言語**: TypeScript, HTML5
- **フレームワーク**: React, Tailwind CSS
- **ホスティング**: GitHub Pages

## 7. 必要な素材・情報 (ユーザー様にご準備いただきたいもの)
1.  **店舗ロゴデータ** (AI, PNG, SVG等)
2.  **写真素材** (外観、内観、メニュー、スタッフなど高画質なもの)
3.  **原稿テキスト** (キャッチコピー、コンセプト文、参加方法など)
4.  **SNSアカウント情報** (リンク先URL: https://x.com/RainyPerch)
